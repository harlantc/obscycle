Searching Within a View

Most views contain a Search for widget which will find the first occurrence of a string in the column selected and scroll to that position in the table. If no match is found, a warning is given. Because of some finicky behavior with the table widget being used in the GUI, you may have to click on the bottom left scrollbar arrow a few times to make your proposal appear or on the right top scrollbar arrow to make you column appear or resize the table. As long as you did not get a warning, the string should be there.

Status Page

When you start the GUI a status page for the panel is displayed. This page records running tallies of edits performed with the GUI. Tallies are red if a balance is overdrawn. You cannot close this window except by quitting the GUI.

Print
Allows you to print the current status of the panel.
Note:
Sometimes the tally does not switch to blue when the balance goes positive. The sign on the number is correct though.

Editing Proposals

The majority of your interactions will be in editing the proposals according to the Panel's wishes. To edit a proposal, you click on the proposal number in the view. This will bring up a notebook with each selected proposal as a tabbed page in the notebook. There are actually very few fields that a panel can edit, and not every proposal contains all of those fields. If a field can be edited, it will be apparent on the Edit Page for the proposal.

You can tab between fields on an edit page (shift-tab goes in reverse). When a change has been made to a field, the tab label is marked with an asterisk when the focus leaves the field being edited. You cannot close a page without dealing with unsaved changes. Closing a page removes it from the notebook. Reset sets all values to their last saved values.

If there are fields that the panel would like to change, but the Edit Page does not allow it, you can put their request in the comments section or contact CDO if that is not sufficient for the panel chair.

Reverse TOO
Some proposals have been designated as reverse TOOs. This designation is assigned by CDO. Such proposals are docked as one TOO trigger based on the target's rev_too value.
